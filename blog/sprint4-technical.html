Problem Solving

4 December 2020 | S4-Tech

Story-Time

When I got blocked on a simple problem.

There was one point I remember I got stuck during kata. The excercise was JS Gradebook. In the excerise you were given data which included student names and separately their scores/grades. What you had to do, was use your knowledge of Javascript to manipulate the given data, and add to a gradebook. The gradebook has to be a separate object or group of data.

I got stuck on the first few tasks, and I realised because I started too complex. I tried to find the most effective way to write the program, with the least amount of code. Ended up using up too much time. Which made me feel so frustrated. So I back-tracked and started again keeping it simple, solving through the error messages. I learned that it was better to make the program work first, and then refactor afterwards.

When I solved a problem in an elegant way.

This excercise was the very last kata exercise involing 'Built-in Methods'. The function I had to write was called 'deBee'. I felt like because it was the last one, I had built up enough momentum, and confidence through practice. Enough so, that my Pseudocode felt more natural, as I knew all the steps that I needed to take. Even if I didn't know what methods to use.

This made me feel very capable and I feel like all these hours of hard work and practice was worth it. I learned that through practice, and doing something every day brings confidence in myself and my abilities.
